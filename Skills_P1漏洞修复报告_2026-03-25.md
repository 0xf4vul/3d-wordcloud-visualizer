### 2. innerHTML 潜在 XSS 风险 ✅

**问题描述**：
- 代码使用 `innerHTML` 插入用户输入的词云数据
- 已通过 `escHtml()` 函数进行转义，但缺少明确的安全注释
- 可能导致代码审查时误认为存在 XSS 漏洞

**修复内容**：

1. **在 escHtml() 函数附近添加安全注释**
   ```javascript
   // HTML 转义函数（P1 XSS 防护）
   // 安全措施：所有用户输入在插入 DOM 前必须通过此函数转义
   // 防止 XSS（跨站脚本）攻击，转义特殊字符：< > " &
   function escHtml(s) {
       return String(s).replace(/&/g, '&amp;')
                     .replace(/</g, '&lt;')
                     .replace(/>/g, '&gt;')
                     .replace(/"/g, '&quot;');
   }
   ```

2. **在 updateStats() 函数中添加安全注释**
   ```javascript
   // 更新统计面板（XSS 防护）
   // 安全措施：使用 escHtml() 转义所有用户输入的词云数据
   // 防止恶意词库包含 <script> 标签或 HTML 代码
   function updateStats(words) {
       const el = document.getElementById('stats-list');
       const html = words.map((w, i) =>
           `<div class="stats-row">
               <span class="stats-rank">${i+1}.</span>
               <span class="stats-word">${escHtml(w.word)}</span>
               ...
           </div>`
       ).join('');
       el.innerHTML = html;
   }
   ```

3. **在 doSnap() 函数中添加安全注释**
   ```javascript
   // 生成磁吸图标
   // 安全措施：label 和 hint 均为硬编码常量，不涉及用户输入
   // 因此此处使用 innerHTML 是安全的，无需 escHtml() 转义
   const icon = document.createElement('div');
   icon.className = 'snap-icon';
   const label = pid === 'upload' ? '📤' : '📊';
   const hint  = pid === 'upload' ? '上传文件' : '查看统计';
   icon.innerHTML = `<span class="si-emoji">${label}</span><span class="si-text">${hint} · 双击恢复</span>`;
   ```
